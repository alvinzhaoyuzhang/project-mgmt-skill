# 架构:双 Base 体系 + 项目级表隔离(B1)

## 决策

两层隔离:

1. **Base 级隔离 · 按业务方向**:每个业务方向一个工作区 Base(例:AI 产品开发、行政合规、市场活动)
2. **表级隔离 · 按项目**:在工作区 Base 内,**每个项目独立一张任务表 + 独立一个综合看板**(B1 方案)

```
通用模板 Base(永久保留 · skill 的 bootstrap 源)
  ├── 项目主表(模板,空)
  ├── 任务表(模板,字段/视图就位,记录空)
  ├── 综合看板(模板,5 block 结构,data_config 待改 table_name+filter)
  └── 速查卡

    │ clone(一次性,按业务方向拆分工作区)
    ▼
工作区 Base · 例如「AI 产品开发」
  ├── 项目主表 ← N 行,每行 = 1 个项目
  ├── 任务表·项目A      ┐
  ├── 任务表·项目B      │  每个项目 1 张独立任务表
  ├── 任务表·项目C      ┘
  ├── 综合看板·项目A    ┐
  ├── 综合看板·项目B    │  每个项目 1 个独立 dashboard
  ├── 综合看板·项目C    ┘
  └── 速查卡(全工作区共享)
```

## 为什么是 B1(同 Base 多任务表),不是 A 或 B2

| 诉求 | A(共享任务表) | **B1** | B2(每项目独立 Base) |
|---|---|---|---|
| PM 跨项目总览 | ✅ 同表天然 | ✅ 同 Base 切表 | ❌ 要切 Base |
| 成员串台防护 | ⚠️ 靠权限行级过滤 | ✅ 物理隔离 | ✅ 物理隔离 |
| 新项目设置成本 | 1 行项目主表 | 克隆 1 张表 + 1 个 dashboard(skill 可自动) | 克隆 1 个 Base + 改名 |
| 统一权限配置 | ✅ 一次配 | ✅ 一次配(Base 级) | ❌ 每 Base 配 |
| 字段演进同步 | 不需要 | 需在 N 张表同步(skill 批量) | 需在 N 个 Base 同步 |
| Base 表数量上限(~20-50) | 无影响 | ⚠️ 10+ 项目可能顶 | 无影响 |
| 跨项目总览 | 天然 | 得建索引表 | 需 Master Base + API 同步 |

**B1 的定位**:中小规模(1-10 个同时进行的项目)的最佳平衡点。

## 模板的两层 clone

**Base 级 clone(业务方向级,低频)**:
- 通用模板 Base → 新工作区 Base("行政合规"、"市场活动" 等)
- 用 `lark-cli base +base-copy`
- 一次性动作,一个业务方向只做一次

**表级 clone(项目级,高频)**:
- 工作区 Base 已有的 任务表(模板) + 综合看板(模板) → 新项目的任务表+dashboard
- 流程:
  1. 读模板任务表字段定义(`+field-list`)
  2. 在工作区 Base 用 `+table-create --fields <schema>` 建新表,名 `任务表·项目X`
  3. 逐个克隆视图(`+view-create`)
  4. 克隆综合看板(新建 dashboard 并逐个建 block,指向新表名)
  5. 在项目主表加 1 行,在 `所属项目` link 字段关联该项目

- Skill 封装后:**一条 `+new-project 项目名` 命令完成上述全流程**

## 什么情况下要改架构

| 触发 | 迁移 |
|---|---|
| 项目数超过 10 个(接近 Base 表数上限) | B1 → B2(分拆到多个工作区 Base 或独立 Base) |
| 跨业务方向项目要合并看板 | 建 Master Base 做 API 同步 |
| 单项目保密等级特别高(需物理隔离) | 该项目单独开 B2 式 Base |

## 风险与备选

**风险 1**:字段演进时 N 张表同步繁琐
- 缓解:skill 提供 `+add-field-to-all-tasks-tables` 批量命令

**风险 2**:项目主表的反向 link 字段 会随项目数增加(每项目一个反向 link)
- 预案:反向 link 命名 `项目任务·项目X`,保持可识别;项目删除时也删对应 link

**风险 3**:Base 的表数量上限
- 飞书限额:单 Base 免费版 20 张,高级版 50 张
- 按"1 项目 = 1 任务表 + 1 dashboard 视图"算,免费版约 10 项目上限
- 超过再迁移到 B2,skill 届时提供迁移工具

## 与 skill 的关系

skill 封装后主要能力:

1. **Bootstrap**:建通用模板 Base + 建工作区 Base(两层 clone)
2. **项目管理**:`+new-project` 一键建任务表+dashboard+权限
3. **字段维护**:`+add-field-to-all-tasks-tables` 批量演进 schema
4. **成员管理**:`+assign-member` 给指定项目的角色绑定成员
5. **规范校验**:检查任务层级合规、完成率异常等

skill 不直接管理业务数据内容,只负责"基础设施铺设 + 规范校验"。
