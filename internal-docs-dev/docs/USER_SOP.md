## 0. 开始之前

### 0.1 这个工具是什么

「项目管理中心」是一个跑在 OpenClaw 里的 skill,会在你的飞书租户内自动搭建一套**经过验证的项目管理模板**:

- 标准化的任务表(支持 4 级层次:目标 → 里程碑 → 工作包 → 任务)
- 综合看板(自动生成进度图表,无需手工配)
- 3 级权限角色(超管 / 项目经理 / 成员)
- 跨项目总览
- 成员邀请、任务跟踪、阻塞预警等开箱即用

它**不**是一个 SaaS — 所有数据都在你自己的飞书租户里,我们只是帮你"按最佳实践搭好骨架"。

### 0.2 你需要什么

| 需要 | 说明 |
|---|---|
| ✅ 飞书账号 | 个人版/企业版均可 |
| ✅ OpenClaw 已连接飞书 | 通过飞书机器人扫码授权,你应该已经做过 |
| ✅ OpenClaw 飞书插件已安装 | 见下方"如何安装飞书插件" |
| 🟡 飞书租户允许建多维表格 | 99% 默认允许,如禁用需联系 IT |
| 🟡 团队成员的飞书账号 | 可后续添加,首次安装可选 |

#### 如何安装飞书插件

skill 启动时会自动检测,如果未安装,在 OpenClaw 终端运行:

```
npx -y @larksuite/openclaw-lark install
```

约 30 秒装好,装好后回到 skill 输入 `r` 重试即可。

### 0.3 它会创建什么

skill 会在你的飞书租户内创建:

- **1 个项目空间**(命名形如「项目管理中心·产品研发」)
- **3 张表**:项目主表 / 任务表(模板)/ 速查卡(规范说明)
- **1 个综合看板**(5 个图表 block 自动展示进度)
- **3 个角色**(超管 / PM / 成员)+ 高级权限规则
- **9 个视图**(全部任务 / 我的任务 / 甘特图 / 阻塞延期 等)

✅ **不会做的事**:
- 不会邀请任何成员(直到你明确选择"邀请")
- 不会创建任何具体项目(直到你跑 `pm new-project`)
- 不会读取你的聊天记录、文档、邮件
- 不会修改你已有的飞书内容

### 2.4 录入第一批任务的最佳实践

新项目建好后,任务表是空的。建议按以下顺序录入:

1. **先建 🎯 目标(1 个)**:写清"项目验收时,什么算完"
   - 例:"Q3 新版本如期上线,核心 5 功能验收通过"
2. **再建 🏁 里程碑(3-7 个)**:把整个项目分成阶段
   - 例:"需求冻结" → "Beta 发布" → "GA 发布"
3. **每个里程碑下建 📦 工作包(2-5 个)**:按模块或角色拆
   - 例:里程碑"Beta 发布"下:前端开发 / 后端开发 / 测试 / 文档
4. **每个工作包下建 📋 任务(1-5 天可完成)**:具体可执行的事
   - 例:"实现 API /v2/users" / "写用户登录测试用例"

**WBS 编号**手动维护,推荐:
- 1 (目标) → 1.1, 1.2, 1.3 (里程碑) → 1.1.1, 1.1.2 (工作包) → 1.1.1.1 (任务)

按 WBS 编号升序排序,任务自然形成树状层次。

### 2.5 邀请团队的最佳时机

✅ **建议**:项目结构搭好(任务录入完成)再邀请成员,成员一进来看到的是清晰的树状任务,而不是空表。

❌ **不建议**:Base 刚建好就邀请,成员看到空表会困惑"这是什么?"。

如果跳过了邀请通知(2.2 选了 [2]),后续:

```
pm share-project --project Q3新版本上线
```

会触发飞书通知,让所有成员收到加入消息。

## 3. PM 日常使用

> 假设你是 **王经理**,负责 2 个项目「Q3 新版本上线」和「客户 A 交付」。

### 3.1 早上 5 分钟巡检

每天打开飞书,按下面顺序看 3 个东西,5 分钟搞定:

#### 步骤 1 · 综合看板(每个项目 1 分钟)

打开「Q3新版本上线 · 综合看板」,**5 个图表只关注 3 个**:

| 图表 | 你要在这看到 |
|---|---|
| 时间压力分布 | "已过期" 这一条有没有变高?变高 = 红灯 |
| 一级工作完成率 | 哪个工作包拖后腿?完成率 < 30% 的最值得关注 |
| 任务状态(按级别) | 📋 二级工作的"阻塞"数 ≥ 3?需要进入下一步排查 |

#### 步骤 2 · 阻塞与延期视图(2 分钟)

切到任务表 → 「阻塞与延期」视图。**看 3 件事**:

1. 哪些任务在阻塞列?**问负责人原因**(直接 @ 飞书消息)
2. 哪些任务延期超过 3 天?**评估是否要重排里程碑**
3. 哪些是 P0/P1 任务?优先处理

#### 步骤 3 · 我负责的项目(1 分钟)

打开项目主表 → 「我负责的项目」视图。
**确认每个项目的"状态"字段是当前真实状态**(进行中 / 阻塞 / 完成),如果跟你认知不符,直接改。

### 3.2 站会上手指南

每周站会拿什么打开?

| 站会主题 | 打开视图 |
|---|---|
| **进度全览** | 综合看板 → 所有图表过一遍 |
| **逐人对齐** | 任务表 → 按"负责人"分组的临时视图 |
| **里程碑追踪** | 任务表 → 「里程碑看板」视图 |
| **本周到期** | 任务表 → 「本周到期」视图 |

**站会模板**(15-30 分钟):
1. 5 分钟:看综合看板,讲整体进度数字
2. 15 分钟:逐成员同步,问 3 个问题:
   - 上次站会到现在做完了什么?
   - 接下来 3 天要做什么?
   - 有没有阻塞?
3. 5-10 分钟:讨论阻塞 + 调整任务

### 3.3 周报怎么用看板自动生成

PM 写周报时,直接截 4 张图:

1. 综合看板 - 时间压力分布(展示风险)
2. 综合看板 - 一级工作完成率(展示进度)
3. 任务表「阻塞与延期」视图(展示问题清单)
4. 任务表「里程碑看板」视图(展示阶段达成)

附上 3 段文字:
- 本周亮点(已完成里程碑/重大交付)
- 风险与阻塞(从阻塞视图列出 Top 3)
- 下周计划(本周到期 + 关键任务)

### 3.4 调整任务分工(成员忙不过来时)

发现 **张三**手上有 8 个进行中任务,**赵五**只有 2 个,需要重新分配:

1. 任务表 → 按"负责人"分组临时视图
2. 找张三那一组的任务,挑出 2-3 个不紧急的
3. 把这些任务的"负责人"字段改为赵五
4. 飞书自动通知赵五

或者更粗暴:

```
pm rebalance --project Q3新版本上线
```

skill 会显示当前每人负载,让你交互式调整。

### 3.5 处理阻塞和延期

#### 阻塞流程

成员把任务标"阻塞"+ 填"风险与阻塞"字段后,你应该:

1. **24 小时内**:打开该任务,看清阻塞原因
2. **判断你能否解开**:
   - 是依赖问题 → 推动依赖方,或调整任务顺序
   - 是资源/人力 → 申请支援或重排
   - 是技术难点 → 找技术 lead 决策
3. **更新任务状态**:
   - 解开了 → 改回"进行中"
   - 依旧阻塞 > 3 天 → 升级到上级,改优先级 P0

#### 延期流程

任务"实际完成日期" > "计划完成日期" → 自动判定为延期。处理:

1. 评估**为什么延期**:估错? 范围扩张? 阻塞?
2. 如果是范围扩张 → **拆任务**(把额外工作拆出新任务)
3. 如果是估错 → **改"计划完成日期"为新预期**
4. 如果是阻塞 → 切换为阻塞流程

### 3.6 项目状态升级

项目主表的"状态"字段:筹备 → 进行中 → 完成 / 取消

**手动升级时机**:

| 当前 → 目标 | 触发条件 |
|---|---|
| 筹备 → 进行中 | 第一个 📋 任务进入"进行中"状态时 |
| 进行中 → 完成 | 所有 🏁 里程碑完成 + 验收通过 |
| 进行中 → 取消 | 项目终止决定,任务表保留作历史归档 |

**自动监控**:综合看板的"任务状态(按级别堆叠)"可以让你一眼看到:
- 🎯 目标 / 🏁 里程碑 全部"已完成" + 但项目主表"状态"还是"进行中" = 该升级了

## 4. 成员日常使用

> 假设你是**张三**,Q3 新版本上线项目的成员。

### 4.1 早上看「我的任务」

打开任务表 → 「我的任务」视图(已按"状态"分组,"计划完成日期"升序)。

**看 3 件事**(2 分钟):

1. **「进行中」组的最上面那条** = 今天的优先级最高任务
2. **「未开始」组的最早到期任务** = 今天可能要启动的
3. **「阻塞」组**:有就赶紧处理或汇报

### 4.2 更新任务进展(每天/每完成一步)

打开你正在做的任务,**填两个字段**:

#### "进度"字段(0-100)

按粗略估计填,不用纠结精确:
- 0 = 没动
- 25 = 已开始
- 50 = 一半
- 75 = 大半好了
- 90 = 收尾
- 100 = 完成(同时把"状态"改"已完成")

#### "最近更新"字段

**推荐格式**:
```
2026-04-25 完成了 API 接口实现 + 单测 80% 通过
下一步:补齐异常分支测试 + 联调
需要:后端确认 /v2/users 字段定义
```

3 句话:**做完啥 / 下一步 / 需要谁**。让 PM 一眼就懂。

### 4.3 把任务标完成

任务真的做完时:

1. "状态" 改为"已完成"
2. "实际完成日期" 选今天
3. "进度" 拉到 100
4. **"交付物"字段填上**:文档链接 / PR 链接 / 演示视频 / 部署链接 等

**为什么要填交付物?** 让 PM 验收时不用问你"在哪看",直接点链接。

### 4.4 遇到阻塞怎么标

如果任务卡住超过 4 小时:

1. "状态" 改为"阻塞"
2. **"风险与阻塞"字段填**:具体阻塞原因 + 需要谁帮忙
   ```
   等待 后端 提供 /v2/users 字段定义。
   已 @后端组长,预计明天 EOD 拿到定义后可继续。
   ```
3. **直接 @ 项目经理**(飞书消息)同步阻塞情况

PM 会在 24 小时内介入。

### 4.5 我请假/出差/离职

#### 请假/出差(< 1 周)

不需要做啥,你的任务自动出现在「我的任务」视图,回来继续。
出差期间如果有人要问你的任务,他们能从你的任务详情里看到"风险与阻塞"和"最近更新"。

#### 长期请假/离职

需要 **PM 介入**:
- PM 把你的所有"进行中"任务的"负责人"字段改成接手人
- "协作人"字段加上你做交接确认
- "最近更新"字段填一段交接说明:你做到哪、上下文是什么

## 5. 维护操作手册

> 所有维护操作都是**幂等的** —— 跑两次结果一样,不用怕重试。

### 5.1 加新项目

```
pm new-project
```

进入 4 题流程(同 [2.1](#21-加新项目流程4-题))。

**注意点**:
- 新项目的任务表是从模板克隆的,你 wizard 时改过模块标签,新项目会沿用最新版
- 如果你升级过 schema(加了字段),新项目自动包含新字段

### 5.2 加成员到现有项目

```
pm add-member --project Q3新版本上线
```

交互式选择:

```
要加什么角色的成员?
  [1] 项目经理(可改任务、协作人、状态)
  [2] 项目成员(只能改自己的任务)

▸ 2
```

```
搜索成员姓名/邮箱:
▸ 王明
```

```
找到:[1] 王明(开发)
确认加 王明 为「Q3新版本上线」的项目成员? [y/n]
```

确认后**自动绑角色 + 分享 Base + 触发飞书通知**。

### 5.3 移除成员

```
pm remove-member --project Q3新版本上线 --member 王明
```

会做 3 件事:
1. 把"负责人 = 王明"的任务,**改为留空**(等你重新分配)
2. 把"协作人"里包含王明的,**移除王明**
3. 解除 王明 在该项目的角色绑定

⚠️ **不会**删除王明已经做完的任务记录(历史保留)。

### 5.4 加字段(全项目)

要给**所有项目的任务表**加一个"客户类型"字段:

```
pm add-field --to all-tasks --name "客户类型" --type select --options "TOC,TOB"
```

skill 会:
1. 列出当前所有任务表(可能 5-10 张)
2. 让你确认要加的字段定义
3. 串行给每张表加字段
4. 验证

如果只想加到某个项目:

```
pm add-field --to project Q3新版本上线 --name "客户类型" ...
```

### 5.5 改字段选项(加状态值等)

例:给"状态"字段加一个新值"待评审":

```
pm edit-field-options --field 状态 --add 待评审
```

skill 会**遍历所有任务表**,把同名字段的选项都更新一遍。

⚠️ **不能改字段类型**(如 select 改 text)— 这种破坏性操作 skill 拒绝执行,需要你手动建新字段然后迁移数据。

### 5.6 删除字段(危险)

```
pm delete-field --field "客户类型" --from all-tasks
```

⚠️ **强警告**:
- 字段删除后,所有相关数据**永久丢失**(飞书回收站不保留字段级删除)
- 跑命令时 skill 会要你输 `DELETE` 大写确认
- 有引用关系的字段(被 lookup/formula 引用)skill 会拒绝删除,要你先解开引用

### 5.7 归档项目

项目结束想归档(保留数据但不再活跃):

```
pm archive-project --project 客户A交付
```

会做:
1. 项目主表中该项目的"状态"改为"已归档"
2. **不删除**任务表,但任务表名前缀加 "[已归档]"(便于识别)
3. **不删除**综合看板,但 dashboard 名加 "[已归档]"
4. 解除所有成员的角色绑定(他们看不到了)
5. 仅超管可访问

要重新激活:

```
pm reactivate-project --project 客户A交付
```

### 5.8 删除项目(危险)

```
pm delete-project --project 客户A交付
```

⚠️ **强警告**:
- 任务表 + dashboard + 项目主表行 全部删除
- 飞书回收站保留 15 天,过期数据**完全丢失**
- skill 强制要求:
  1. 先 `pm archive-project` 归档(冷却期)
  2. 至少 7 天后才能 `pm delete-project`(避免误操作)

### 5.9 升级 schema(skill 出新版)

skill 自身可能升级(比如加了新 formula 字段或新视图)。升级时:

```
pm upgrade
```

skill 会:
1. 对比线上 schema vs skill 当前版本的 schema
2. 显示差异清单(加了什么、改了什么)
3. 让你逐项确认
4. 串行升级所有任务表
5. 验证

**例**:skill v1.1 加了"复盘日期"字段。升级时:

```
📦  schema 升级 v1.0 → v1.1

变更:
  + 任务表加字段「复盘日期」(datetime)
  + 综合看板加 block「按月任务量」(列表 9 个旧空间需要更新)

将影响:
  - 4 张任务表(加字段)
  - 4 个综合看板(加 block)

继续? [y / n / 看详情]
▸ 
```

### 5.10 健康检查

跑命令:

```
pm verify
```

或

```
pm verify --base <空间名>
```

输出:

```
🔍  健康检查

  ✅  3 张表齐全
  ✅  字段完整
  ✅  3 角色齐全
  ✅  综合看板 5 block 齐全
  
  ⚠️  发现 1 项风险:
      任务表「客户A交付」有 5 个 📋 任务无负责人
      → 跑 pm find-orphan-tasks 查清单
```

**建议每周跑一次** 作为体检。

## 6. 故障排查 · FAQ

### 6.1 网络/连接问题

#### "网络无法访问飞书"

```
❌  网络无法访问飞书
```

**排查步骤**:
1. 浏览器打开 `https://open.feishu.cn` —— 能打开吗?
   - 不能 → 公司网络/VPN 屏蔽,联系 IT
2. 你在 VPN 里吗?
   - 是 → 关 VPN 重试
3. 飞书自身宕机?查 status.feishu.cn

### 6.2 权限问题

#### "401 Unauthorized" 或 "token 失效"

**原因**:OpenClaw 的飞书登录 token 过期了。

**修复**:回 OpenClaw,通过飞书机器人重新连接 → 重试 skill。

#### "403 Forbidden" 或 "无权限"

**原因**:
1. 你不是该 Base 的超管 → 联系空间所有者
2. 飞书租户禁用了多维表格写权限 → 联系 IT

### 6.3 字段冲突 / 重复

#### "字段名重复:任务名称"

**原因**:wizard 之前中断遗留,部分字段已建。

**修复**:
1. 选 [r] 重试,skill 会自动跳过已存在的字段
2. 或选 [u] 撤销,完全删除空间从头来过

### 6.4 飞书表数量上限

#### "已达单 Base 表数量上限(20)"

**原因**:你在多任务表方案下加了太多项目。

**修复路径**:
1. 归档老项目(`pm archive-project`)→ 不会真删表但减少视觉负担
2. 升级到多空间方案(`pm migrate-to-multi-spaces`)→ 拆分到多个 Base

### 6.5 飞书 API 限速

#### "Rate limit exceeded"

**原因**:短时间内调用太多 API。

**自动处理**:skill 自动等 60 秒后重试。
**手动**:把 wizard 暂停,15 分钟后再续跑。

### 6.6 数据丢失恢复

#### "我误删了一个任务"

**飞书回收站(15 天)**:
1. 打开任务表 → 工具栏 → 回收站
2. 找到该记录 → 还原

#### "我误删了一整张任务表"

**同样 15 天回收站**:
1. 飞书工作台找到该 Base → 右键 → 历史版本
2. 选删除前的版本恢复

#### "我误删了整个空间"

**Base 级回收站(15 天)**:
1. 飞书工作台 → 回收站
2. 找到该 Base → 还原

### 6.7 wizard 中途挂了

#### "上次状态文件损坏"

**修复**:
```
pm init --reset
```

会清空本地状态从头开始。
飞书内已建的部分需要手动删除(或用 `pm clean` 清残留)。

### 6.8 想完全卸载 skill

```
pm uninstall
```

会:
1. 列出 skill 创建过的所有空间
2. 让你逐个选保留 / 删除
3. 删除本地缓存(~/.pm-skill/)

⚠️ **不会自动删空间**,得你逐个确认 — 防止误操作。

### 6.9 高级用户 · 导出全部数据备份

```
pm export --base <空间名> --output backup.zip
```

导出:
- 所有表的 CSV
- 所有 dashboard 的 JSON 配置
- 所有视图的 JSON 配置
- 角色 + 权限规则

可用于离线备份或迁移到其他飞书租户。

### 6.10 跑命令时报 "lark-cli: command not found"

**原因**:OpenClaw 的飞书插件没装好。

**修复**:

```
npx -y @larksuite/openclaw-lark install
```

装好后回 OpenClaw 重试 skill。

### 6.11 我的飞书是"个人版"(非企业租户),能用吗?

可以,但有限制:
- ✅ 多维表格创建、字段、视图、dashboard 全部可用
- ⚠️ "高级权限"功能在个人版可能受限,3 角色权限隔离可能弱化
- ⚠️ Base 表数量上限可能更严(免费版 20 张)

建议:个人版玩耍 OK,企业用建议升级到付费租户。
