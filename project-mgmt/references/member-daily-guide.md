# 成员日常使用引导

> 当用户(身份是项目成员)问"我每天该做什么"时读这个文件。

## 早上看「我的任务」

打开任务表 → 「我的任务」视图(已按"状态"分组,"计划完成日期"升序)。

**看 3 件事**(2 分钟):

1. **「进行中」组的最上面那条** = 今天的优先级最高任务
2. **「未开始」组的最早到期任务** = 今天可能要启动的
3. **「阻塞」组**:有就赶紧处理或汇报

## 更新任务进展(每天 / 每完成一步)

打开你正在做的任务,**填两个字段**:

### "进度"字段(0-100)

按粗略估计填,不用纠结精确:

- 0 = 没动
- 25 = 已开始
- 50 = 一半
- 75 = 大半好了
- 90 = 收尾
- 100 = 完成(同时把"状态"改"已完成")

### "最近更新"字段

**推荐格式**:

```
2026-04-25 完成了 API 接口实现 + 单测 80% 通过
下一步:补齐异常分支测试 + 联调
需要:后端确认 /v2/users 字段定义
```

3 句话:**做完啥 / 下一步 / 需要谁**。让 PM 一眼就懂。

## 把任务标完成

任务真的做完时:

1. "状态" 改为"已完成"
2. "实际完成日期" 选今天
3. "进度" 拉到 100
4. **"交付物"字段填上**:文档链接 / PR 链接 / 演示视频 / 部署链接 等

**为什么要填交付物?** 让 PM 验收时不用问你"在哪看",直接点链接。

## 遇到阻塞怎么标

如果任务卡住超过 4 小时:

1. "状态" 改为"阻塞"
2. **"风险与阻塞"字段填**:具体阻塞原因 + 需要谁帮忙

```
等待 后端 提供 /v2/users 字段定义。
已 @后端组长,预计明天 EOD 拿到定义后可继续。
```

3. **直接 @ 项目经理**(飞书消息)同步阻塞情况

PM 会在 24 小时内介入。

## 我请假 / 出差 / 离职

### 请假 / 出差(< 1 周)

不需要做啥,你的任务自动出现在「我的任务」视图,回来继续。
出差期间如果有人问你的任务,他们能从任务详情里看到"风险与阻塞"和"最近更新"。

### 长期请假 / 离职

需要 **PM 介入**:
- PM 把你的所有"进行中"任务的"负责人"字段改成接手人
- "协作人"字段加上你做交接确认
- "最近更新"字段填一段交接说明:你做到哪、上下文是什么

## 怎么找到不属于自己的任务参考

切到「全部任务(树状浏览)」视图(权限会自动过滤,你只看到有权访问的)。
按 WBS 编号排序,可以理解整个项目结构。

## skill 回复样式建议

当用户问成员视角问题:
- 给精准短答案(成员要的是"今天我该干嘛",不是长篇大论)
- 引用具体字段名(进度 / 最近更新 / 交付物)
- 不要假设技术背景(用户可能是非技术成员)
