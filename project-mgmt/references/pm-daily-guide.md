# PM 日常使用引导

> 当用户(身份是项目经理)问 "我每天打开应该看什么 / 怎么用这个空间" 时读这个文件。
> 给出**具体场景化的引导**,不要泛泛而谈。

## 早上 5 分钟巡检

按下面顺序看 3 个东西,5 分钟搞定。

### 步骤 1 · 综合看板(每个项目 1 分钟)

打开「<项目名> · 综合看板」,**5 个图表只关注 3 个**:

| 图表 | 看什么 |
|---|---|
| **时间压力分布** | "已过期"那条有没有变高?变高 = 红灯 |
| **一级工作完成率** | 哪个工作包拖后腿?完成率 < 30% 的最值得关注 |
| **任务状态(按级别)** | 📋 二级工作的"阻塞"数 ≥ 3?需要排查 |

### 步骤 2 · 阻塞与延期视图(2 分钟)

切到任务表 → 「阻塞与延期」视图。**看 3 件事**:

1. 哪些任务在阻塞列?**问负责人原因**(直接 @ 飞书消息)
2. 哪些任务延期超过 3 天?**评估是否要重排里程碑**
3. 哪些是 P0/P1 任务?优先处理

### 步骤 3 · 我负责的项目(1 分钟)

打开项目主表 → 「我负责的项目」视图。
**确认每个项目的"状态"字段是当前真实状态**(进行中 / 阻塞 / 完成),如果跟你认知不符,直接改。

## 站会上手指南

每周站会拿什么打开?

| 站会主题 | 打开视图 |
|---|---|
| **进度全览** | 综合看板 → 所有图表过一遍 |
| **逐人对齐** | 任务表 → 按"负责人"分组的临时视图 |
| **里程碑追踪** | 任务表 → 「里程碑看板」视图 |
| **本周到期** | 任务表 → 「本周到期」视图 |

**站会模板**(15-30 分钟):
1. 5 分钟:看综合看板,讲整体进度数字
2. 15 分钟:逐成员同步,问 3 个问题:
   - 上次站会到现在做完了什么?
   - 接下来 3 天要做什么?
   - 有没有阻塞?
3. 5-10 分钟:讨论阻塞 + 调整任务

## 周报怎么用看板自动生成

PM 写周报时,直接截 4 张图:

1. 综合看板 - 时间压力分布(展示风险)
2. 综合看板 - 一级工作完成率(展示进度)
3. 任务表「阻塞与延期」视图(展示问题清单)
4. 任务表「里程碑看板」视图(展示阶段达成)

附上 3 段文字:
- **本周亮点**(已完成里程碑/重大交付)
- **风险与阻塞**(从阻塞视图列 Top 3)
- **下周计划**(本周到期 + 关键任务)

## 调整任务分工

发现某成员手上任务过多,需要重新分配:

1. 任务表 → 按"负责人"分组临时视图
2. 找过载成员那一组,挑 2-3 个不紧急的
3. 把这些任务的"负责人"字段改为接手人
4. 飞书自动通知接手人

## 处理阻塞和延期

### 阻塞流程

成员把任务标"阻塞"+ 填"风险与阻塞"字段后,你应该:

1. **24 小时内**打开该任务,看清阻塞原因
2. **判断你能否解开**:
   - 是依赖问题 → 推动依赖方,或调整任务顺序
   - 是资源/人力 → 申请支援或重排
   - 是技术难点 → 找技术 lead 决策
3. **更新任务状态**:
   - 解开了 → 改回"进行中"
   - 依旧阻塞 > 3 天 → 升级到上级,改优先级 P0

### 延期流程

任务"实际完成日期" > "计划完成日期" → 自动判定为延期。处理:

1. 评估**为什么延期**:估错? 范围扩张? 阻塞?
2. 如果是范围扩张 → **拆任务**(把额外工作拆出新任务)
3. 如果是估错 → **改"计划完成日期"为新预期**
4. 如果是阻塞 → 切换为阻塞流程

## 项目状态升级

项目主表的"状态"字段:筹备 → 进行中 → 完成 / 取消

**手动升级时机**:

| 当前 → 目标 | 触发条件 |
|---|---|
| 筹备 → 进行中 | 第一个 📋 任务进入"进行中"状态时 |
| 进行中 → 完成 | 所有 🏁 里程碑完成 + 验收通过 |
| 进行中 → 取消 | 项目终止决定,任务表保留作历史 |

**自动监控**:综合看板的"任务状态(按级别堆叠)"可以让你一眼看到:
- 🎯 目标 / 🏁 里程碑 全部"已完成" + 但项目主表"状态"还是"进行中" = 该升级了

## skill 触发时的回复样式建议

当用户问 PM 视角的问题:
- 不要把整个文档丢给他
- 选 1-2 个最相关章节(早巡 / 站会 / 周报)
- 简短给出步骤,问他"你今天遇到的具体场景是什么"
- 根据回答深入(比如"成员卡住了"→ 走阻塞处理流程)
